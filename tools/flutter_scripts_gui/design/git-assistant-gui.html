Matched · Push rejected

Non-fast-forward push to main

Why: Your local branch and the remote have diverged. Remote has commits you do not have, so a normal push would overwrite history and is blocked.

Safe fix available Avoid force-push on main GitHub · branch protection
STEP 1 Confirm with diagnose
STEP 2 Apply the fix
STEP 3 Verify push works

1. Diagnose first

Run these to confirm what the model saw matches your repo. 

git fetch origin
git status -sb

Updates remote refs, then shows whether you are ahead, behind, or diverged — without changing your files. 

git log --oneline --left-right HEAD...origin/main

Lists commits only on your side vs only on the remote so you can see the diverge clearly.

2. Solution

Do not use git push --force on protected main. Prefer rebase (or merge) then a normal push. 
git pull --rebase origin main
git push origin HEAD

Replays your commits on top of the latest remote main, then pushes. If conflicts appear, resolve files and run git rebase --continue.

How-to

Commit changes

Stage what you intend to keep, then create a local commit with a clear message. Nothing is shared until you push.

  1. See what changed

    git status
git diff

    Lists modified files; diff shows the exact line changes before you stage.

  2. Stage and commit

    git add -p
git commit -m "Describe why, not only what"

    -p lets you stage hunks intentionally so you do not commit leftovers by accident.
